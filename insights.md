# 🧠 Personal Insights & Tips

## 1. 프레임워크와 사고의 구조화

프레임워크나 의사결정트리 같은 구체적인 방법론을 요구하면 AI가 사고의 구조를 잡아주어 내용의 누락을 방지하고, 훨씬 논리적인 결과물을 만들어냅니다.

### 예시

* **[프롬프트 기초] 프롬프트 프레임워크** : Persona, Task, Context, Format을 활용해 모든 프롬프트의 기본 뼈대를 구성합니다.
* **[교육 콘텐츠] 멘탈 모델 & 의사결정 트리** : 학습자의 사고 과정을 설계하고 논리적인 흐름을 구축할 때 강력한 힘을 발휘합니다.
* **[이력서/경력] 3C4P, START 기법** : 자신의 성과를 구조화하고 전문성을 극대화하여 경력 기술서를 작성할 때 최적화된 도구입니다.


## 2. 실존 인물 페르소나의 활용

페르소나 설정 시 추상적인 '전문가'라는 표현보다 구체적인 **실존 인물**을 언급하는 것이 AI로부터 더 깊이 있고 통찰력 있는 답변을 유도하는 데 효과적입니다.

### 예시

* **[서비스 기획] 폴 그레이엄 (Paul Graham)** : 스타트업의 본질을 꿰뚫는 간결하고 강력한 비즈니스 로직을 점검할 때 활용합니다.
* **[재테크/경영] 김승호 회장** : '돈의 속성'에 근거한 사업가적 마인드셋과 자본 관리 조언을 얻기에 유용합니다.
* **[관계/소통] 마셜 로젠버그 (Marshall Rosenberg)** : 비폭력 대화(NVC) 전문가로서 공감 중심의 갈등 해결 가이드를 제공받습니다.
* **[프론트엔드] 켄트 C. 도츠 (Kent C. Dodds)** : AHA Programming("추상화보다 복사 붙여넣기가 낫다") 철학의 창시자. 베스트 프랙티스 가이드 작성 시 '나쁜 코드 → 문제 발견 → 리팩토링 → 베스트 프랙티스'로 이어지는 단계별 진화 과정을 시각화하는 데 활용합니다.
* **[프론트엔드] 조쉬 코모 (Josh W. Comeau)** : 눈에 보이지 않는 개념(CSS 렌더링, React 동작 원리 등)을 멘탈 모델 기반으로 설명하는 데 특화. "이 기술은 이런 그림으로 돌아갑니다"처럼 시각적 비유와 도식으로 개념을 이미지화할 때 활용합니다.


## 3. 모델 확장을 위한 메타 프롬프팅

익숙하지 않은 새로운 작업을 시작할 때는 AI에게 직접 해당 분야에 적합한 모델이나 인물을 추천받아 시스템을 점진적으로 확장할 수 있습니다.

새로운 작업 전, 아래와 같은 질문을 통해 최적의 페르소나와 프레임워크를 먼저 확보하세요.

```
내가 지금 [특정 작업]을 하려고 해. 이 분야에서 가장 뛰어난 통찰을 줄 수 있는 실존 인물 페르소나 3명과, 결과물의 논리 구조를 잡아줄 수 있는 프레임워크 2개를 추천해주고 그 이유를 설명해줘.
```


## 4. 답변 품질을 높이는 프롬프트 팁

응답의 정확도를 높이고 대화의 흐름을 효율적으로 관리하기 위해 특정 문구를 전략적으로 배치합니다.

* **정확도 향상** : 작업을 계속하기 전에 후속 조치나 명확한 질문을 요청하여 AI가 맥락을 더 깊이 이해하도록 유도합니다.

    ```작업을 계속하기 전에 후속 조치나 명확한 질문을 해주세요```

* **컨텍스트 관리** : 응답 품질이 저하될 때 기존 내용을 요약한 뒤, 이를 바탕으로 새 채팅을 시작하여 성능을 초기화합니다.

    ```지금까지의 대화를 요약해줘```
